# Imagility

Imagility 是一款面向 Windows 桌面的图片批量处理工具，基于 WPF 和 .NET Framework 4.8 构建。它把图片压缩、格式转换、尺寸调整、水印、背景消除、证件照换底色、拼接拼贴、PDF 归档、画笔打码和手写签名叠加集中在一个本地应用里完成。

## 主要功能

- **批量导入**
  - 支持选择多张图片。
  - 支持拖拽图片或文件夹导入。
  - 支持 JPG、JPEG、PNG、BMP、GIF、WebP、TIFF 等常见格式。
  - 新导入图片默认不勾选，导入后自动预览本次新增的第一张图片。

- **压缩与格式转换**
  - 支持保持原格式、JPEG、PNG、WebP、BMP、GIF、TIFF。
  - 支持有损/无损模式。
  - 支持质量调节。
  - 支持按目标大小压缩，适用于普通图片导出和拼接输出。
  - 目标大小以 KB 输入，JPEG/WebP 会自动试探质量；不适合目标大小压缩的格式会自动转为 JPEG。
  - 如果处理结果包含透明背景，会优先使用支持透明通道的 WebP，避免透明信息丢失。
  - PNG 支持调色板量化和颜色数量控制。

- **尺寸调整**
  - 支持像素、厘米、毫米、英寸单位。
  - 支持 DPI 换算。
  - 支持保持比例。
  - 切换图片后自动识别当前原图尺寸。

- **背景消除**
  - 独立负责识别纯色背景并处理为透明背景。
  - 支持识别容差和边缘过渡控制。
  - 同时启用证件照处理时，会按证件照主背景识别，降低黑框、相框和边缘干扰。

- **证件照处理**
  - 独立负责证件照换底色。
  - 支持蓝底、白底、红底、浅灰底和自定义底色。
  - 支持从图片内缩边缘估算证件照主背景色，避免把外部黑框或相框误识别为背景。
  - 支持清理背景水印、压缩噪点、暗角等小瑕疵区域。
  - 可单独使用，也可与背景消除组合使用：先抠除背景，再铺设证件照底色。

- **文字水印**
  - 支持自定义水印文字。
  - 支持位置、字号、透明度设置。

- **合并、拼接与拼贴**
  - 支持纵向拼接、横向拼接、网格拼贴、瀑布流拼贴、主图海报、自由排列。
  - 支持列数、间距、画布尺寸、外边距、填充方式和背景色设置。
  - 瀑布流拼贴会遵循当前填充方式，例如完整留白、填满裁切或拉伸。
  - 勾选 2 张及以上图片时，右侧“编辑后预览”可显示拼接/拼贴后的效果。

- **PDF 归档**
  - 支持合并为单个 PDF。
  - 支持每张图片输出一个 PDF。
  - 支持页面 DPI 设置。
  - 默认 96 DPI，适合屏幕 100% 查看；可手动设置 300 DPI 用于打印归档。
  - PDF 内嵌图片按当前压缩质量生成，不按“目标大小”逐页试探。
  - 导入图片后可直接导出 PDF，不需要先进行其他编辑或转换。
  - 不勾选图片时默认导出全部图片；有勾选时只导出勾选图片。

- **画笔打码**
  - 支持马赛克和高斯模糊。
  - 支持画笔粗细、马赛克块大小、模糊半径设置。
  - 每张图片保留独立打码历史。
  - 支持撤销上一步打码。
  - 打码坐标按原图方向统一处理，预览、批量导出、拼接和 PDF 输出保持一致。

- **手写签名**
  - 支持新建透明背景签名 PNG。
  - 支持鼠标、触摸或手写笔绘制。
  - 支持画笔粗细调整、撤销、清空、保存。
  - 保存签名时可选择 PNG 保存路径，默认目录为用户应用数据目录下的 Imagility 签名目录。
  - 支持将签名插入图片，并在预览区拖拽移动、滚轮缩放。
  - 每张图片保留独立签名状态。
  - 批量导出、拼接、PDF 导出均会保留签名叠加效果。

## 界面逻辑

- 左侧列表显示已导入图片。
- 左侧列表高亮表示当前正在预览的图片。
- 复选框表示批量处理范围。
- 导入图片默认不勾选，避免误批量处理。
- “开始转换”优先处理已勾选图片；如果没有勾选图片，则处理当前正在预览的图片。
- “合并/拼接”需要至少勾选 2 张图片。
- “导出 PDF”在未勾选图片时默认导出全部已导入图片；如果有勾选，则只导出勾选图片。
- 右侧“编辑后预览”显示当前参数处理后的效果。
- 勾选 2 张及以上图片时，右侧预览会自动切换为拼接/拼贴效果；取消到少于 2 张后回到当前单图编辑预览。
- 快速切换图片、参数或预设色时，旧预览请求不会覆盖新的预览结果。

## 项目结构

```text
Photo-zip/
├─ Assets/                         # 图标和 Logo 资源
├─ Converters/                     # WPF 绑定转换器
├─ Models/                         # 图片项、打码、签名状态模型
├─ Services/                       # 图片处理、拼接、PDF 写入核心逻辑
├─ Properties/                     # 传统 .NET 项目属性资源
├─ App.xaml                        # 全局样式和应用入口资源
├─ MainWindow.xaml                 # 主界面
├─ MainWindow.xaml.cs              # 主界面交互和状态编排
├─ SignatureWindow.xaml            # 签名绘制窗口
├─ SignatureWindow.xaml.cs         # 签名绘制和保存逻辑
└─ Photo-zip.csproj                # WPF 项目文件
```

## 构建

在项目根目录执行：

```bash
dotnet restore "Photo-zip.csproj"
dotnet build "Photo-zip.csproj" -c Release
```

默认输出路径：

```text
bin/Release/net48/Photo-zip.exe
```

如果需要避免共享编译进程影响验证，可以使用：

```bash
dotnet build "Photo-zip.csproj" --no-restore -c Release -p:UseSharedCompilation=false -v:minimal
```

## 基本使用

1. 打开应用。
2. 点击“添加图片”，或把图片/文件夹拖入窗口。
3. 按需要设置格式、质量、目标大小、尺寸、水印、背景消除、证件照底色、打码、签名或拼接参数。
4. 如果要批量处理，勾选需要处理的图片；如果只处理当前预览图，可以不勾选。
5. 点击“开始转换”“合并/拼接”或“导出 PDF”。
6. 点击“打开输出文件夹”查看结果。

## 输出规则

- 默认输出到原图目录下的 `Converted` 文件夹。
- 可手动选择输出目录。
- 文件名冲突支持覆盖、自动重命名、跳过。
- 如果输出目录就是原图目录，且输出路径会与源文件完全相同，应用会避免直接覆盖源文件。
- 单张转换完成后提示“处理完成”，多张转换完成后提示“批量处理完成”。
- PDF 合并模式默认输出 `archive.pdf`。

## 开发说明

- `Services/ImageProcessingService.cs` 是核心处理层，负责图片转换、预览、拼接、PDF 写入、打码和签名合成。
- `MainWindow.xaml.cs` 负责界面状态、参数收集、预览刷新和任务调度。
- 普通转换、拼接和 PDF 会在任务开始时固定本次图片状态，避免任务运行中继续编辑影响同一批输出。
- 图片处理链路会统一处理 EXIF 方向，避免手机竖图出现预览和导出方向不一致。
- 背景消除、证件照处理、目标大小编码和 PDF 页面生成支持取消请求。
- 构建产物 `bin/`、`obj/` 已通过 `.gitignore` 排除。
